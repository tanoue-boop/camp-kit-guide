# タイトル・メタディスクリプション パターン集

`affiliate-seo` スキルの一部。タイトルと description の設計に迷ったときに参照する。

---

## タイトル設計の原則

### 字数

- **理想：30〜35 字**
- 上限：60 字（検索結果での切り詰めを避ける）
- 主要 KW を **先頭から 30 字以内** に配置

### 含めるべき要素

```
[主要KW] + [商品数] + [年号] + [副訴求]
```

- **主要KW**：検索ボリュームのある中核キーワード
- **商品数**：「おすすめ 5 選」など具体数で網羅性を訴求
- **年号**：【YYYY年版】で鮮度シグナル
- **副訴求**：ターゲット読者に響く差別化フレーズ

---

## 記事タイプ別タイトル雛形

### おすすめ N 選型（最も汎用）

```
{ジャンル/カテゴリ}おすすめ{N}選【{年}年版】{差別化}
```

例：
- `キャンプ用ナイフおすすめ5選【2026年版】初心者でも安心の選び方`
- `ファミリーキャンプ用ランタンおすすめ5選【2026年版】家族みんなで快適な夜を`

### 比較型

```
{商品A} vs {商品B}徹底比較！{差別化軸}で選ぶおすすめ
```

例：
- `モンベル vs スノーピーク 徹底比較！コスパで選ぶキャンプギア`

### 選び方ガイド型

```
{ジャンル}の選び方完全ガイド｜{N}つのポイントと失敗しないコツ
```

例：
- `テントの選び方完全ガイド｜4つのポイントと失敗しないコツ`

### レビュー型

```
{商品名}レビュー｜{使用シーン/期間}で分かった{要点}
```

例：
- `モーラナイフ コンパニオン レビュー｜半年使って分かった魅力と注意点`

### Q&A 型（情報検索系）

```
{疑問形KW}？{結論を含む副題}
```

例：
- `キャンプナイフは違法？知っておくべき法律と安全な持ち運び方`

## description 設計の原則

### 字数

- **120〜150 字**を目標
- 上限：160 字（モバイル検索結果での切り詰めを避ける）

### 構成パターン

```
[誰に][何を] + [どう解説するか] + [何が分かるか]
```

### テンプレート

```
{ターゲット}向けに{ジャンル}を{視点}で解説。
{選び方の軸}の{N}点で比較し、{得られる価値}を紹介。
{追加情報}も解説。
```

### 良い例（camp-knife-beginner より）

> キャンプ用ナイフを初めて選ぶ方向けに、刃の素材・刃長・収納方式・安全性の 4 点で選び方を解説。モーラナイフやオピネルなど定番ブランドの中から初心者でも失敗しにくい 5 製品を厳選し、用途別の選び方とお手入れのコツも紹介。

このフォーマットの分析：

- 冒頭でターゲット（初心者）を明示
- 選び方の 4 軸を具体的に列挙（KW包含）
- 紹介する商品の例を 2 ブランド名で具体性
- 「用途別の選び方」「お手入れのコツ」で記事の網羅性をアピール

### NG パターン

- 100 字未満（情報量不足）
- 160 字超（切り詰められる）
- KW を冒頭に含めない
- 「〜について解説します」で抽象的に終わる
- 記事内容と乖離した煽り文句

## 検索意図とタイトル・description の整合

KW から想定される検索意図と、タイトル・description が応えている内容が一致していること。

| KW タイプ | 検索意図 | タイトル雛形 |
|---|---|---|
| `{ジャンル} おすすめ` | 比較して選びたい | おすすめN選型 |
| `{ジャンル} 選び方` | 判断軸を学びたい | 選び方ガイド型 |
| `{商品名} レビュー` | 使った人の感想 | レビュー型 |
| `{ジャンル} 違法／法律` | 安全性確認 | Q&A型 |
| `{商品A} {商品B}` | 比較中 | 比較型 |
| `{ジャンル} 初心者` | 入門需要 | おすすめN選型（初心者訴求を副題に） |

---

## 年号運用

- `【YYYY年版】` を title・description 双方に入れる
- 公開時の年に揃える。年明けには記事タイトル・description・本文の年号更新を推奨
- 過去年号のままにしておくと「古い情報」として CTR が下がる

---

## title と description の重複回避

- description に title をそのままコピーしない
- title で示した結論の **理由・詳細** を description で補完する関係性が理想

---

## サイトとの整合（Seo.tsx の挙動）

camp-kit-guide では `Seo.tsx` が以下を自動処理している：

```tsx
const fullTitle = title.includes(SITE_NAME) ? title : `${title} | ${SITE_NAME}`;
```

つまり title プロパティに SITE_NAME を含めなければ自動で末尾に `| サイト名` が付く。各サイトで同じパターンを採用する場合、frontmatter の title にはサイト名を含めなくてよい。
