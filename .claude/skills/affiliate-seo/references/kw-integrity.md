# KW 整合性ルール

`affiliate-seo` スキルの一部。検索キーワードと記事内商品リストの整合性を確保するためのルール。

**重要：このファイルは原則のみを定義する。具体的な数値（価格レンジの幅、ブランド重複の上限件数など）は商材ごとに最適値が異なるため、各サイトの `CLAUDE.md` で定義する。**

---

## なぜ KW 整合性が重要か

検索 KW から流入したユーザーは、KW が示す **暗黙の期待**を持って記事を読み始める。

- 「キャンプナイフ 初心者」で来た人 → 安価で扱いやすいモデルを期待
- 「キャンプナイフ ハイエンド」で来た人 → 高品質・高価格帯を期待

紹介する商品リストがこの期待とズレると：

- 直帰率上昇 → SEO シグナル悪化
- コンバージョン率低下
- 「期待外れ」の体験 → ブランド毀損

---

## 整合性チェックの 3 軸

### 軸 1：予算帯の整合

KW が暗示する **想定予算** に対し、紹介商品の価格帯が合っていること。

```
KW タイプ → 想定予算 → 商品価格帯
─────────────────────────────────
{ジャンル} 安い        → 平均価格以下
{ジャンル} コスパ      → 平均価格以下〜中央値
{ジャンル} 初心者      → 想定予算 ±α 以内
{ジャンル} おすすめ    → 平均価格中央あたり
{ジャンル} ハイエンド  → 中位〜高価格帯
{ジャンル} プロ用      → 高価格帯
```

**サイトごとに定義すべき数値**：
- 「初心者 KW」での価格レンジ許容幅（±X%）
- 「コスパ KW」での価格上限（平均価格の Y% 以内）
- 商材カテゴリごとの想定予算ベンチマーク

### 軸 2：レベル感の整合

KW が暗示する **ユーザー層・用途のレベル感** に合った商品を選定。

| KW シグナル | 想定ユーザー | 商品選定方針 |
|---|---|---|
| 「初心者」「入門」「最初の」 | 初購入者 | 操作簡単・定番ブランド・低リスク |
| 「コスパ」「安い」 | 価格重視 | 平均以下の価格で機能十分 |
| 「ファミリー」「家族」 | 家族利用 | 安全性・大容量・複数人対応 |
| 「ソロ」「一人」 | 単独利用 | 軽量・コンパクト・シンプル |
| 「ハイエンド」「プロ」「上級者」 | 経験者 | 高機能・高品質・高価格帯OK |
| 「2026 年最新」 | 最新志向 | 直近 1〜2 年の新モデル中心 |

### 軸 3：商品多様性の確保

同一記事内で **同一ブランドへの偏りを避ける** こと。理由：

- 読者に複数選択肢を提示する記事の本旨を満たす
- ブランド推し記事と判断されると検索順位に悪影響
- 利用規約で広告主単独推薦を制限するアフィリエイトプログラムもある

具体的な上限件数（同一ブランド最大 N 件まで等）は商材により最適値が異なるため、各サイトの `CLAUDE.md` で定義する。

---

## 公開前 KW 整合性チェック

記事公開前に以下を確認：

- [ ] **記事タイトルの KW** から想定される予算帯と、紹介商品の価格帯が乖離していないか
- [ ] **記事タイトルの KW** から想定されるレベル感（初心者/コスパ/ハイエンド等）と、各商品の実態が乖離していないか
- [ ] **5 商品中、同一ブランドの偏りがないか**（具体上限はサイトの CLAUDE.md 参照）
- [ ] **商品リスト内の最高価格と最低価格の差** が極端でないか（具体倍率はサイトの CLAUDE.md 参照）
- [ ] **商品の実在確認**：架空のブランド・モデルでないか
- [ ] **代表的・人気モデル**：マイナーすぎる商品ばかりで構成されていないか

---

## 不整合の典型パターンと対処

### パターン A：「初心者 KW」なのに高価格モデルが混入

**例**：「キャンプナイフ初心者おすすめ 5 選」で 5 万円の高級ナイフを紹介

**対処**：
- 価格帯を初心者層の想定予算（数千円〜1.5 万円程度）に絞る
- どうしても高価格モデルを含める必要があるなら、タイトルを「初心者から上級者まで」に変更

### パターン B：「コスパ KW」なのに高価格商品が混入

**例**：「コスパ最強ランタン 5 選」で 2 万円のモデルが含まれる

**対処**：
- コスパ KW では商品リスト全体が **平均価格以下** に収まるよう厳選
- 高品質高価格モデルは別記事「ハイエンドランタン」「長く使える本格派」等で紹介

### パターン C：同一ブランド偏重

**例**：5 商品中 3 商品がスノーピーク

**対処**：
- ブランド多様性を確保（最大件数はサイトの CLAUDE.md に従う）
- 「スノーピーク特集」として記事のスコープを変更するなら可

### パターン D：価格帯の極端な乖離

**例**：最低価格 2,000 円 / 最高価格 50,000 円（25 倍差）

**対処**：
- 同一記事内の価格レンジを縮める（具体倍率はサイトの CLAUDE.md）
- 高価格帯を別記事に分離

## サイト固有設定（必須）

このスキルを採用する場合、各サイトの `CLAUDE.md` に以下を定義する：

```md
## KW 整合性ルール（{サイト名} 固有値）

### 価格レンジ許容幅
- 初心者 KW：想定予算の ±__% 以内
- コスパ KW：商品リスト全体が平均価格の __% 以下
- ハイエンド KW：中位以上のみ

### 商品リスト内の価格倍率
- 5 商品の最高価格 ÷ 最低価格 = __ 倍以内

### 同一ブランドの上限
- 1 記事内で同一ブランド最大 __ 件まで
（または「多様性を確保すること」のみ記載で運用判断）

### 商材カテゴリごとの想定予算ベンチマーク
- カテゴリ A：初心者 ¥__〜¥__ / 中位 ¥__〜¥__ / 上位 ¥__以上
- カテゴリ B：...
```

## このルールは AI による記事生成にも適用

Claude Code 等で記事を AI 生成する場合も、このルールに従って商品選定をする。AI が学習知識ベースで商品リストを作るとブランド偏重や価格乖離が起きやすいので、以下を併用：

1. **商品候補は外部 API（楽天等）の実データから**選ぶ
2. **AI が選定した結果を人間がレビュー**してから公開
3. **公開前 KW 整合性チェック** を必ず実施
