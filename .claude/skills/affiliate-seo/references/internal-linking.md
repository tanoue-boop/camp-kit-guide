# 内部リンク戦略

`affiliate-seo` スキルの一部。記事内・サイト内の内部リンクを設計する際に参照する。

---

## なぜ内部リンクが重要か

1. **回遊深度の向上**：1 ユーザーあたりの PV 増加 → 滞在時間延長 → SEO シグナル改善
2. **コンバージョン機会の拡大**：複数記事を読むほどアフィリリンク露出が増える
3. **クロール効率**：検索エンジンの bot がサイト構造を把握しやすくなる
4. **テーマの結びつき**：関連ページを相互リンクすることでトピックの権威性が伝わる

---

## 3 つの内部リンク軸

### 軸 1：記事内アンカー（商品ID ↔ まとめ表）

**最重要パターン**。記事末尾の「用途別おすすめ表」から各商品ブロックへのアンカーリンクで戻る。

実装：

```mdx
{/* 商品ブロックには id を付与 */}
<ProductCardMdx
  rank="1"
  id="mora-companion"
  ...
/>
```

```md
| 優先したいこと | おすすめモデル | 価格帯 |
|---|---|---|
| コスパ重視の入門ナイフ | [モーラナイフ コンパニオン](#mora-companion) | 約2,750円 |
```

### CSS 補正（固定ヘッダ対策）

```css
/* アンカー遷移時、固定ヘッダで隠れないように */
.productAnchor {
  scroll-margin-top: 90px;
}
```

または React 側でラップ：

```tsx
<div id={id} style={{ scroll-margin-top: "90px" }}>
  <ProductCard ... />
</div>
```

### id 設計ルール

- **記事内で一意**（重複 NG）
- 半角英数字とハイフンのみ
- 商品ブランド名 + モデル名で意味のある形に（例：`mora-companion`、`opinel-no8`）
- スラッグ生成関数を用意して機械的に生成しても可

---

### 軸 2：カテゴリ内回遊（関連記事）

同カテゴリの他記事を 3 件程度自動表示する。

実装ロジック例：

```tsx
// getStaticProps 内
const relatedPosts = allPosts
  .filter((p) => p.slug !== currentSlug && p.category === currentCategory)
  .slice(0, 3);
```

表示位置：

- **記事末尾**（まとめの後）が最も読まれる
- まとめ→関連記事→次記事への自然な流れ

### 関連記事の選定基準

優先度が高い順：

1. **同カテゴリ内で公開日が近い記事**（鮮度）
2. **同カテゴリ内で人気の記事**（閲覧数）
3. **タグの重複度が高い記事**

複数基準を組み合わせると質が上がる。Supabase 等で閲覧数を取得できれば 2. を採用する価値あり。

---

### 軸 3：パンくずリスト

```
ホーム > {カテゴリ名} > {記事タイトル}
```

役割：

1. 上位階層への戻り導線
2. BreadcrumbList スキーマでの構造化データ
3. ユーザーの現在地把握

実装：

- 記事ページの最上部（H1 の上）に配置
- カテゴリ名は実カテゴリページへのリンク
- 末尾の記事タイトルはリンクなし（現在ページ）

## 補助的な内部リンク

### 文脈リンク（contextual link）

本文中で関連記事を自然に紹介する。

```md
焚き火台と一緒に使うナイフの選び方は[こちらの記事](/posts/camp-knife-beginner)
で詳しく解説しています。
```

### グローバルナビ

ヘッダーの主要カテゴリ一覧。全ページから全カテゴリページへ最短 1 クリックで到達できるように。

### サイドバー

長文記事ページ：

- 人気記事 TOP5
- カテゴリ一覧（記事数付き）
- 最新記事

これらは内部リンク強化と回遊促進を同時に達成する。

---

## アンカーテキストの原則

リンクのテキスト（アンカーテキスト）は **遷移先のページ内容を表す** こと。

| ❌ 避ける | ⭕ 推奨 |
|---|---|
| 「こちら」 | 「キャンプ用ナイフの選び方」 |
| 「詳しくはこの記事」 | 「初心者向けナイフ 5 選」 |
| 「click here」 | 「ベストセラーランタン比較」 |

ただし、本文の自然な流れを壊さない範囲で。**100% KW 一致のアンカーばかり** は不自然なのでバランスを見る。

---

## アフィリエイトリンクの取り扱い

外部のアフィリエイトリンクは内部リンクとは別の扱い：

- `rel="nofollow sponsored"` を付与（プラットフォーム要件に従う）
- `target="_blank" rel="noopener"` で別タブ
- 内部リンクとは目的が違う（収益のため）ので戦略を分離して考える

## サイト構造（情報設計）

理想的なサイト構造：

```
ホーム
├─ カテゴリページ（一覧）
│   ├─ 記事 A
│   ├─ 記事 B
│   └─ 記事 C
└─ ランキングページ（横断）
```

Hub-and-Spoke モデル：

- **Hub**：カテゴリ TOP・トピッククラスター記事（網羅的な選び方ガイド）
- **Spoke**：個別商品レビュー、特定軸の比較記事

Hub から Spoke、Spoke から Hub への双方向リンクを意識する。

---

## サイト固有設定

各サイトの `CLAUDE.md` で以下を定義：

- カテゴリ slug 一覧
- カテゴリ間の関係性（親子カテゴリがあれば）
- パンくずの実装ファイル
- 関連記事の取得ロジック・件数
- 人気記事の閲覧数取得元（Supabase 等）
