# 信頼シグナル

`affiliate-uiux` スキルの一部。アフィリ記事を「広告ではなく信頼できる情報」と感じさせる UI/UX 要素。

---

## なぜ信頼シグナルが重要か

- アフィリ記事は読者が **「これは広告か？」と警戒** している前提で設計する
- 警戒を解くには **客観的根拠の可視化** が最も効く
- 信頼が確保されればコンバージョン率が大きく改善
- E-E-A-T シグナルとも連動し、SEO にも好影響

---

## レビュー数の明示表示

### ❌ 評価のみ
```
★★★★☆ 4.5
```

### ⭕ 評価 + レビュー数
```
★★★★☆ 4.5（3,200 件）
```

レビュー数が多いほど評価の信頼性が高まる。**3 桁区切り**で具体感を出す。

### 複数プラットフォームの併記

楽天と Amazon の両方で評価されている商品は、両方表示することで信頼性が倍増：

```
Amazon  ★ 4.5（3,200 件）
Rakuten ★ 4.4（  980 件）
```

各プラットフォームのアイコンを併記するとさらに視認性が上がる。

### 0 件レビューは表示しない

レビュー数 0 件の場合は **その行を非表示** にする。「★ 0.0（0 件）」と表示するとむしろマイナス。

```tsx
{rakutenReviewCount > 0 && (
  <div className="rating-row">
    <RakutenIcon />
    ★ {rakutenRating.toFixed(1)}（{rakutenReviewCount.toLocaleString()} 件）
  </div>
)}
```

これは構造化データ（aggregateRating）の付与ルールとも整合する。

## ランキングバッジ（金/銀/銅）

### 1〜3 位は装飾強め

```css
.rank1 { background: linear-gradient(135deg, #f59e0b, #d97706); }  /* 金 */
.rank2 { background: linear-gradient(135deg, #94a3b8, #64748b); }  /* 銀 */
.rank3 { background: linear-gradient(135deg, #d97706, #92400e); }  /* 銅 */
```

メダル風の配色を使うことで、ユーザーが直感的に「上位」を理解できる。

### 4 位以下は控えめに

順位を明示する場合も、4 位以下は文字のみ・背景色なし等で控えめに：

```css
.rank4plus {
  background: var(--color-card-bg);
  color: var(--color-text-sub);
  font-weight: 600;
}
```

### 配置

絶対配置で商品カードの top-left に配置。視線の流れ（左上から右下）に沿って真っ先に目に入る。

## 推し訴求バッジ

差別化フレーズを商品名の上に配置。1 商品 1 バッジ程度に絞る：

```
[コスパ最強・初心者No.1]
モーラナイフ コンパニオン
```

### スタイル

```css
.badge {
  display: inline-block;
  background: #fef3c7;
  color: #92400e;
  font-size: 0.875rem;
  font-weight: 700;
  padding: 0.25rem 0.5rem;
  border-radius: 6px;
}
```

薄い黄色 + 焦げ茶のコントラストで「特別感」を演出。

### バッジ文言の作り方

- **対象 + 形容**：「初心者向け」「コスパ重視」
- **No.1 訴求**：「初心者 No.1」「コスパ最強」
- **シーン特化**：「ファミリー向け」「ソロキャンプ向け」

過度な誇張（「絶対買うべき」「最強」連発）は避け、対象や用途を絞った訴求に。

## 比較表での視覚的強調

### 第 1 行（最高位商品）はハイライト

```css
.row-first {
  background: linear-gradient(
    to right,
    rgba(245, 158, 11, 0.05),
    transparent
  );
}
```

薄い金色のグラデで「これがおすすめ」を視覚的に伝える。

### 偶数行はゼブラ

```css
.row-even {
  background: var(--color-card-bg);
}
```

スキャン読みやすさを向上。

### 「○」マークは SVG チェックマークに変換

`○` `✓` `◎` `yes` `true` `✔` 等の真偽値マーカーは、SVG チェックマークに自動変換すると視認性が向上：

```tsx
const TRUE_VALUES = ["○", "✓", "◎", "yes", "true", "✔"];

function renderCell(value: string) {
  if (TRUE_VALUES.includes(value.toLowerCase())) {
    return <CheckIcon className="text-green-600" />;
  }
  return value;
}
```

## CTA ボタンの色 = プラットフォームのブランドカラー

### 心理的効果

ユーザーは普段使う EC サイトの色を覚えている。同じ色を使うことで：

- 「公式と連動している」感
- ボタンの認識スピードが上がる（迷わずクリック）
- 信頼感の向上

### 実装例

```css
.cta-amazon  { background: #FF9900; color: #FFFFFF; }
.cta-rakuten { background: #BF0000; color: #FFFFFF; }
.cta-yahoo   { background: #FF0033; color: #FFFFFF; }
.cta-booking { background: #003580; color: #FFFFFF; }
.cta-airbnb  { background: #FF5A5F; color: #FFFFFF; }
.cta-agoda   { background: #FF6B00; color: #FFFFFF; }
.cta-klook   { background: #FF5722; color: #FFFFFF; }
```

### CTA テキスト

「ボタンを押すと何が起きるか」を明示：

- ⭕ 「Amazon で購入する」「楽天で見る」
- ❌ 「クリック」「こちら」

## 価格の信頼性

### 「最新価格」表示

```
¥19,800（2026/04/28 時点）
```

- 価格は変動するため取得日時を明示
- 「最終更新日」付きで誠実さを訴求

### 価格 ¥ マーク

```
¥{Number(price).toLocaleString()}
```

3 桁区切りで桁数を瞬時に把握できる。

### 価格帯の明示

「予算別 おすすめ」のような表で、価格帯を範囲で見せる：

```
| 用途 | 商品 | 価格帯 |
|---|---|---|
| 初心者向け | モーラナイフ | 約 2,750 円 |
| デザイン重視 | オピネル | 約 2,200 円 |
```

---

## 更新日明示

### 記事メタ情報

```
公開日：2026/04/20
更新日：2026/04/25
```

- 古い情報のまま放置されていない安心感
- BlogPosting スキーマの `dateModified` と同期する

### 更新時の対応

価格・スペック・推奨モデルが変わったら：

- 該当箇所を更新
- frontmatter の `updatedAt` を今日の日付に
- 「2026/04/25 に最新情報に更新しました」等の注記を冒頭に置くのも良い

## PR / 広告表記（信頼性の核）

### 配置

- 記事冒頭の **アイキャッチ直下** または **H1 直後**
- 商品紹介セクション直前にも繰り返し可

### 文言例

```
※本記事はアフィリエイトプログラムを利用しています
※本ページにはプロモーションが含まれています
※当サイトは Amazon アソシエイト・プログラムに参加しています
```

### スタイル

控えめだが見える：

```css
.affiliate-disclaimer {
  font-size: 0.875rem;
  color: var(--color-text-sub);
  text-align: center;
  padding: 0.5rem;
  background: var(--color-card-bg);
  border-radius: 6px;
}
```

### 法的要件

- 景品表示法・ステマ規制対応の必須要素
- 「分かりやすい場所に明示」が義務
- 隠す・小さくしすぎる等は規制違反

## 出典・引用元の明示

### 本文中の引用

```
> メーカー公式情報によると、刃の素材はステンレス鋼...
> （[引用元](https://...)）
```

### 統計・データ

```
2026 年のキャンプ用品市場規模は前年比 105%（出典：日本アウトドア協会調べ）
```

数字を出すときは出典を必ず添える。

---

## 著者情報・運営者情報

### 著者情報の表示位置

- 記事冒頭または末尾に「執筆：{著者名}」
- クリックで著者プロフィールページへ
- アイコン付きだと視認性が上がる

### 運営者情報

- フッターから「運営者情報」「運営会社」へリンク
- About ページに：
  - 運営会社名・所在地
  - サイトの目的・編集方針
  - 著者一覧
  - 問い合わせ窓口

これらは E-E-A-T シグナルとも直結。

## 信頼シグナル チェックリスト

実装後の確認：

- [ ] レビュー数が件数で明示されている（「★4.5（3,200件）」）
- [ ] レビュー 0 件の行は非表示
- [ ] 1〜3 位に金/銀/銅バッジが配置されている
- [ ] 推し訴求バッジは 1 商品 1 個程度
- [ ] 比較表で最高位商品がハイライト
- [ ] CTA ボタンが各プラットフォームのブランドカラー
- [ ] CTA テキストが「{プラットフォーム}で{動作}」形式
- [ ] 価格に取得日時 or 「最新価格」表示
- [ ] 公開日・更新日が両方表示されている
- [ ] PR / 広告表記が冒頭に明示
- [ ] 出典・引用元が必要箇所で明示
- [ ] 著者情報または運営者情報がアクセス可能

## サイト固有設定

各サイトの `CLAUDE.md` で以下を定義：

- 採用するアフィリエイトプラットフォーム（楽天/Amazon/Booking/Klook 等）
- それぞれの CTA カラーコード
- PR 表記の正確な文言（法令対応の最新版）
- 著者情報の表記方法（個人名 / Organization）
- レビューデータの取得元（API / 手動）
