# ページレイアウト

`affiliate-uiux` スキルの一部。各種ページのレイアウト設計。

---

## 記事ページ（最重要）

### デスクトップ：2 カラム + sticky サイドバー

```
┌─────────────────────────────────┐
│ Header                          │
├─────────────────────────────────┤
│ アイキャッチ（高さ 360px、全幅）  │
├─────────────────────────────────┤
│ ┌──────────────┐ ┌──────────┐ │
│ │              │ │          │ │
│ │   本文        │ │ サイドバー│ │
│ │  (1fr)       │ │ (300px)  │ │
│ │              │ │ sticky   │ │
│ │              │ │          │ │
│ └──────────────┘ └──────────┘ │
├─────────────────────────────────┤
│ 関連記事グリッド（3列）           │
├─────────────────────────────────┤
│ Footer                          │
└─────────────────────────────────┘
```

実装：

```css
.layout {
  display: grid;
  grid-template-columns: 1fr 300px;
  gap: 2rem;
  max-width: 1160px;
  margin: 0 auto;
  padding: 0 1rem;
}

.sidebarWrap {
  position: sticky;
  top: 80px;
  align-self: start;
}
```

### モバイル：1 カラム

960px 以下：

```css
@media (max-width: 960px) {
  .layout {
    grid-template-columns: 1fr;
  }
  .sidebarWrap {
    position: static;
  }
}
```

サイドバーは本文の下に移動。サイドバー TOC は CSS で非表示にし、代わりに記事内のインライン TOC が機能する。

### 本文エリア内の構造

```
パンくずリスト
H1（記事タイトル）
  メタ情報（公開日 / カテゴリ / タグ）
インライン TOC
本文
  H2 / H3
  ProductCard × N
  ComparisonTable
  Tips
  FAQ
  まとめ
ShareButtons
RecommendedPosts（同カテゴリ関連記事）
PostNavigation（前後記事リンク）
```

## アイキャッチエリア

### デスクトップ

```css
.eyecatch {
  height: 360px;
  width: 100%;
}
.eyecatchImg {
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```

### 画像なし時のフォールバック

frontmatter で `thumbnail: ""` の場合、グラデーション背景でフォールバック：

```css
.eyecatchGradient {
  background: linear-gradient(
    135deg,
    var(--color-accent),
    var(--color-accent-light)
  );
  display: flex;
  align-items: center;
  justify-content: center;
}
```

中央にカテゴリ名やサイトロゴを白文字で配置するパターンも可。

### モバイル

```css
@media (max-width: 480px) {
  .eyecatch { height: 240px; }
}
```

## トップページ

### 構造

```
Hero（サイト概要 + 主要訴求）
   ↓
カテゴリグリッド（全カテゴリへの導線）
   ↓
ランキングウィジェット（人気記事 TOP10）
   ↓
最新記事一覧
   ↓
Footer
```

### Hero セクション

- サイトの **30 秒で分かる価値提案**
- メインビジュアル（ライフスタイル感のある画像）
- 主要 CTA ボタン（「カテゴリから探す」「ランキングを見る」等）
- 高さ：50〜70vh 程度（スクロール促進のため画面いっぱいにはしない）

### カテゴリグリッド

- 全カテゴリを一覧表示
- 各カテゴリにアイコン（絵文字または SVG）
- グリッド：PC 3〜4 列 → タブレット 2 列 → モバイル 2 列（または 1 列）
- 各カードはカテゴリページへのリンク

### ランキングウィジェット

- 全カテゴリ横断の人気記事 TOP10
- 順位 + サムネ + タイトル + カテゴリバッジ
- 1〜3 位は装飾強め（金/銀/銅バッジ）

---

## カテゴリページ

```
パンくずリスト（ホーム > {カテゴリ名}）
H1（カテゴリ名）
カテゴリ説明文（〜200字）
記事グリッド（3列、PC）
ページネーション（多数記事の場合）
```

### 記事グリッド

```css
.articleGrid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1.5rem;
}

@media (max-width: 960px) {
  .articleGrid { grid-template-columns: repeat(2, 1fr); }
}

@media (max-width: 600px) {
  .articleGrid { grid-template-columns: 1fr; }
}
```

各記事カード：

- サムネイル（16:9）
- タイトル（2 行で省略）
- 説明文の冒頭（1〜2 行）
- カテゴリバッジ + 公開日

## ランキングページ（横断ランキング）

サイト全体の人気記事を順位付け表示。

```
H1（{年}年最新ランキング）
リード文
  記事 1（順位+大きめ表示）
  記事 2
  記事 3
  ...
```

- 1〜3 位は特に装飾強め
- 順位の変動があれば「↑3」等の変動表示
- 各カードからアフィリリンクへの最短導線

## ヘッダー

### 構造

```
[ロゴ]    [カテゴリナビ]    [検索]
```

- 高さ：60〜80px
- `position: sticky; top: 0;` で常時表示
- スクロール時にも見える
- 背景：白 or 半透明 + ブラー

### モバイル

ハンバーガーメニュー化：

```
[ロゴ]                    [≡]
```

タップで全画面メニューが展開。

## フッター

### 構造

```
┌──────────────────────────────────┐
│ サイト名                          │
│ サイト説明（1〜2 行）              │
│                                  │
│ カテゴリ一覧 | 運営者情報 | 利用規約│
│ プライバシーポリシー | お問い合わせ │
│                                  │
│ © {年} {運営会社名}                │
└──────────────────────────────────┘
```

- 背景：アクセントカラー（または暗い色）、文字白
- リンクは下線なし、hover で下線

### 必須リンク

- 運営者情報（特商法対応）
- プライバシーポリシー
- 利用規約
- お問い合わせ
- 免責事項
- アフィリエイト免責事項（必要に応じて）

## 余白設計の原則

### 縦の余白

- セクション間：3〜4rem
- 段落間：1〜1.5rem
- 見出しと次の要素：0.5rem
- ProductCard 同士：1.5rem

### 横の余白（PC）

- max-width：1160px（記事ページ）/ 1280px（トップ）
- 両端 padding：1rem〜2rem

### モバイル

- 両端 padding：1rem
- セクション間を縮める：2〜3rem

---

## サイト固有設定

各サイトの `CLAUDE.md` で以下を定義：

- max-width 値（サイトのデザイン方針による）
- ヒーローの高さ・コンテンツ
- カテゴリグリッドの列数
- 必須フッターリンクの追加項目（特定の法令対応等）
